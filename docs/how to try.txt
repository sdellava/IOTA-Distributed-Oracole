Latest main quick check
-----------------------

To try the latest main version:

git fetch --all --tags
git checkout main
git pull

Task 0 is the scheduler role. It is enabled for nodes that accept template 0.
Only the node that owns the scheduler queue turn runs the validator-committee
prune, and only when the current IOTA epoch is different from the last pruned
epoch stored in the oracle NodeRegistry. If no scheduled tasks are due, the
node advances the scheduler queue without opening and closing an empty
scheduler round.

open a new terminal and execute:
cd ./node
npm run dev -- --node 1

open a new terminal and execute:
cd ./node
npm run dev -- --node 2

open a new terminal and execute:
cd ./node
npm run dev -- --node 3

open a new terminal and execute a direct one-shot task:
cd ./client
npm run create -- .\examples\task_weather.json

The direct one-shot path uses oracle_tasks::create_and_submit_direct_task. It
creates the task, assigns nodes immediately and submits run 1 without entering
the scheduler queue.

To prepare a scheduled task transaction for the wallet/webview, use:
cd ./client
npm run create -- prepare-task-schedule-webview .\examples\task_weather.json .\examples\schedule_5m_two_runs.json 0xYOUR_WALLET_ADDRESS

Scheduled tasks use oracle_tasks::create_task and are handled by task 0. The
schedule is used only when interval_ms is at least 300000 and the window allows
at least two runs. Otherwise the client prepares a direct one-shot transaction.
Before using the example schedule file, update startAt/endAt to future,
minute-aligned timestamps.

In the webview, leave "Interval (minutes)" empty for a direct one-shot task.
Set an interval only when the task must run more than once.
